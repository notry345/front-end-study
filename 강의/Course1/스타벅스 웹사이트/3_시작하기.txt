> HTML 구조와 범위

태그는 기본적으로 각각의 영역을 가짐

<!DOCTYPE html> 
- DOCTYPE(DTD, Document Type Definition)은 마크업 언어에서 문서 형식을 정의하여
웹 브라우저가 어떤 HTML 버전으로 페이지를 해석하면 되는지 알려주는 용도
- 문서(하나의 페이지)의 HTML 버전을 지정
- 현재 가장 최신 버전이자 웹 표준은 HTML5은 html로 표기
- 그 전 버전인 XHTML 버전으로 만들어진 페이지가 있을 수 있음

<html>
- 시작태그
- HTML 문서의 전체 범위를 브라우저에게 알려주는 역할

    <head>
    - 문서의 정보를 나타내는 범위
    - 브라우저가 해석해야 할 웹 페이지의 제목, 설명, 사용할 파일들의 위치, 스타일 등
    눈에 보이지 않는 정보를 작성하는 범위
    </head>

    <body>
    - 문서의 구조를 나타내는 범위
    - 사용자의 화면을 통해 보여지는 헤더, 푸터, 메뉴, 이미지 등 웹페이지의 보여지는 구조를 작성하는 범위
    </body>

</html>
- 종료태그

> HTML 태그 역할 살펴보기

<title></title> : HTML 문서의 제목, 다른 페이지와 구분할 수 있게 브라우저 탭에 표시됨

<link rel="" href="" /> : 외부 문서(CSS 파일, 로고 등의 이미지 파일)를 가져와 연결할 때 사용
rel(relationship)은 가져올 문서와의 관계, href(hyper reference)는 가져올 문서의 경로

cf) 페이지에 대표로 사용되는 아이콘은 favorite icon을 줄인 favicon(파비콘)으로 부름

<style></style> : 스타일(CSS)를 HTML 문서 안에서 작성하는 경우에 사용
태그 내부에 CSS 문법을 작성하여 적용될 수 있게 함

note) 스타일의 선언 방식은 다양함

<script src=""></script> : src라는 HTML 속성을 사용하여 외부의 자바스크립트 파일을 가져오는 경우
<script></script> : 자바스크립트를 HTML 문서 안에서 작성하는 경우 태그 내부에 자바스크립트 코드 작성

<meta /> : 위의 태그로 나타낼 수 없는 나머지 모든 정보(제작자, viewport 등)들을 표시하여 브라우저와 검색엔진에게 알려줌
<meta name="" content="" />
<meta charset="" />
name은 정보의 종류, content는 정보의 값
charset(Character Set)는 문자 처리 방식(문자인코딩)을 명시하며 UTF-8 주로 사용


> 프로젝트 폴더 관리

다수의 이미지, css, js 파일들을 사용해야 하는 경우 각각을 폴더로 만들어서 관리

BUT index.html 파일은 루트 경로(프로젝트의 최상위 경로)에 위치시켜야 함
브라우저는 프로젝트 폴더에 들어가자마자 index라는 이름의 HTML 파일을 먼저 찾기 때문
-> index.html 파일이 들어있는 폴더 경로만 명시를 해줘도 자동으로 브라우저가 처리

사이트 제작시 메인 페이지는 프로젝트의 루트 경로에 있는 index.html 파일 안에다가 만들고
그 외 페이지들은 별도의 폴더를 만들어서 브라우저가 자동으로 찾을 수 있게 index라는 이름의 HTML 파일로 만들어 주기
해당 하위 페이지에서만 사용하는 CSS, 이미지 파일 등을 폴더 안에 함께 관리

> 이미지와 경로

1. 상대 경로

./ (생략가능) - 현재 파일의 주변에서 경로 찾음
../ - 현재 폴더의 상위폴더로 올라가서 경로 찾음

<img src="./images/logo.png" alt="HEROPY logo">
<img src="images/logo.png" alt="HEROPY logo" />
<!-- alt(Alternate)는 이미지가 출력되지 못하는 경우 대체할 텍스트로 이는 (태그를 완성하기 위해 무조건 작성해야하는) 필수속성-->

2. 절대 경로 : 출발부터 시작해서 도달하는 경로가 모두 일치

http (https) - 원격
/ (//) - 루트경로(최상위경로, 프로젝트 경로로 들어오자마자 있는 위치)

<img src="https://heropy.blog/css/images/logo.png" alt="HEROPY logo">
<!-- 절대 경로, 원격 사이트의 주소로 접근해서 디렉토리 안의 logo.png라는 이미지를 찾아와서 화면에 출력 -->
<img src="/강의/Course1/스타벅스 웹사이트/test/images/logo.png" alt="HEROPY logo" />
<!-- 맨 앞의 슬래시 기호는 프로젝트의 절대 경로이고,
이 슬래시 앞쪽에는 로컬호스트에 현재 개발 서버가 열려져 있는 5500번 포트에 해당하는 주소가 생략되어 있는 것 -->


브라우저 주소창에 http://127.0.0.1:5500/index.html 혹은 localhost:5500
VSCode의 live server가 5500번 포트를 열어서 프로젝트를 출력하고 있는 것
로컬 환경(자신의 컴퓨터, localhost)에서 내부적으로 라이브 서버 같은 플러그인의 도움을 받아 개발 서버를 만들 수 있고 각 개발 서버는 포트 번호에 의해 구별됨


> 링크

<a href=""></a>
href(hypertext reference, 다른 페이지로 이동할 수 있는 참조)는 경로를 입력하는 속성
a 태그 사이에 입력된 내용을 클릭하면 href 속성에 명시되어있는 경로로 페이지를 이동시켜 줌
