요소(element)

<태그>내용</태그>
시작태그 종료태그 사이에 contents가 위치
열린태그 닫힌태그


> 부모와 자식 관계

텍스트 뿐만 아니라 새로운 요소도 콘텐츠로 사용할 수 있음

<태그1>
  <태그2>내용</태그2>
</태그1>

태그1은 부모 요소, 태그2는 자식 요소 <- 상대적인 관계
시각적 구분을 위해 들여쓰기(tab)와 내어쓰기(shift+tab) 적극 활용
코드 정리는 굉장히 중요

<태그>
  <태그>
    <태그>내용</태그>
  </태그>
</태그>

상위(조상) 요소 : 부모 요소, 부모의 부모 요소, ...
하위(후손) 요소 : 자식 요소, 자식의 자식 요소, ...


> 빈 태그(empty tag) : 종료태그가 없음

작성방법1 : <태그> - 편리함, HTML1/2/3/4/5에서 사용
작성방법2 : <태그 /> - 명확, 엄격, 안전함, XHTML/HTML5에서 사용 <- 권장

<br /> <- 줄 바꿈 용도

빈 태그들은 각각의 역할을 명확하게 하기 위해서
많은 경우 속성과 값을 입력하여 추가적인 기능을 확장해서 작성해야함
꼭 필요한 속성을 요소의 필수 속성이라고 함
<img src="" alt="" />
<input type="text" />
등등


> 속성

시작태그에는 속성(attribute)과 값(value)이라는 개념 적용하여 기능의 확장 가능

<태그 속성="값">내용</태그>

기능의 확장이란..
<img src="./image.jpg" alt="blank" />
이미지의 경로나 대체 택스트 등을 명시


> 글자와 상자 : 요소가 화면에 출력되는 특성(CSS와 밀접), 크게 2가지로 구분

1. 인라인(inline) 요소 : 글자를 만들기 위한 요소들, 제약 사항 많은 편
- ex) <span></span>
      본질적으로 아무것도 나타내지 않고 그저 콘텐츠의 영역을 설정하는 용도로 쓰이는 대표적인 인라인 요소
- 요소가 (왼쪽에서 오른쪽으로) 수평으로 쌓임
- span 요소 간 줄바꿈 처리시 하나의 띄어쓰기로 해석되어 요소 사이에 간격이 벌어짐
- 기본적으로 자신이 포함하고 있는 콘텐츠의 크기만큼 가로와 세로 자동으로 줄어듬
- 글자를 취급하는 요소이기 때문에 style 속성으로 width와 height 지정 불가능
- 중요) 글자 요소는 가로, 세로 사이즈를 지정할 수 없다
- 인라인(글자) 요소에는 외부든 내부든 위와 아래 여백이 정상적으로 적용되지 않음, 좌우 여백만 적용됨
- 중요) 인라인 요소에는 자식 요소로 블록 요소 사용 불가, 또 다른 인라인 요소는 가능
        <span><div></div></span> - X
        <span><span></span></span> - O

2. 블록(block) 요소 : 레이아웃(상자)를 만들기 위한 요소들, 제약 사항 크게 X
- ex) <div></div>
      본질적으로 아무것도 나타내지 않고 그저 콘텐츠의 영역을 설정하는 용도로 쓰이는 대표적인 블록 요소
- 요소가 (위에서 아래로) 수직으로 쌓임
- 가로 너비는 부모 요소의 크기만큼 자동으로 늘어남
- 세로 너비는 포함한 콘텐츠 크기만큼 자동으로 줄어듬
- 인라인 요소와 달리 가로, 세로 사이즈 지정 가능
  <div style="width: 100px; height: 40px;">hello</div>
- 외부 여백, 내부 여백을 위, 아래, 좌, 우 모두 자유롭게 설정 가능
  <div style="margin: 10px; padding: 10px;">hello</div>
  margin : 요소의 외부 여백을 지정하는 CSS 속성
  padding : 요소의 내부 여백을 지정하는 CSS 속성
- 중요) 블록 요소는 자식으로 블록 요소와 인라인 요소를 모두 가질 수 있다
        <div><div></div></div> - O
        <div><span></span></div> - O
